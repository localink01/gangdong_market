# 로컬마켓 UI/UX 개선 실행 계획

작성일자: 2026-05-30
문서 버전: v0.1
적용 범위: 로컬마켓 사용자 화면 전반 (운영자 화면은 별도 트랙)

기준 문서
- 로컬마켓 IA 및 URL 규칙 설계안 (v0.1)
- 로컬마켓 화면별 UI/UX 기획서 (v1.0)
- 유즈케이스 카탈로그
- 유즈케이스 상세 명세서

---

## 0. 이 문서의 목적

1. 하단 네비게이션을 포함한 UX 구조 개편을 한 번에 진행하지 않고, 위험을 분산해 단계적으로 적용한다.
2. 네비게이션 외 화면별·인터랙션·정보표현·디자인 시스템 단위로 산재한 개선 사항을 한 곳에 모은다.
3. 각 항목은 변경 규모, 영향 화면, 위험도, 우선순위를 명시해 의사결정과 일정 산정을 쉽게 한다.

---

## 1. 개편 원칙

1. 한 번에 한 가지 인지부하만 바꾼다.  
   라벨 변경, 정보구조 변경, 컴포넌트 변경을 동시에 적용하지 않는다.
2. 모든 변경은 롤백 가능해야 한다.  
   기능 플래그 또는 한 커밋 단위 격리를 기본으로 한다.
3. 변경 후 KPI 검증 항목을 사전에 정의한다.  
   탭별 진입률, 핵심 행동 전환율, 체류시간을 최소 기준으로 본다.
4. 디자인 시스템(토큰·컴포넌트)에 영향이 가는 변경은 design-system 문서에 먼저 반영하고 코드에 적용한다.

## 2. 단계적 로드맵 (3단계)

### 2-1. Phase 1 — 라벨·문구 정비 (저위험, 즉시 가능)

목표
- 사용자 언어로 라벨을 맞추고, 내부 용어를 화면에서 제거한다.

작업 항목
1. 하단 탭 라벨 변경
   - 포털 → 홈
   - 상점가 → 탐색 (적용은 Phase 2 전까지 보류 가능, Phase 1은 "포털 → 홈" 단일 변경만 권장)
2. 화면 헤더 카피 점검
   - M-00 헤더: "지자체 포털" 류 표현 제거, 지자체 브랜딩명 우선 노출
   - MK-00 헤더: "상권 포털" 류 표현 제거, 상권 이름 우선 노출
3. 빈 상태 카피·CTA 점검
   - 비멤버 락 화면 문구를 한 문장으로 통일
   - "멤버십 등록 후 볼 수 있어요" 톤 일관화

영향 화면
- TabBar, RegionPortal, MarketHome, CommunityFeed, BenefitsHub

리스크
- 낮음. 시각적 일관성 영향만 있음.

검증
- 사용자 언어 체크리스트 통과
- 디자인 시스템 카피 가이드 충족 여부

---

### 2-2. Phase 2 — 정보 재배치 (중위험, 점진 적용)

목표
- 커뮤니티·단골 소식의 발견 가능성을 높이고, 혜택의 접근 경로를 마이 안으로 흡수한다.

작업 항목
1. 하단 탭 정의 변경 (Option A 1차 적용)
   - 탭 구성: 홈 / 탐색 / 소식 / 마이
   - "소식" 탭에 community, dangolFeed, dangolPost 화면 매핑
   - "혜택" 탭 제거, MyPage 상단에 "내 쿠폰" / "내 스탬프" 카드를 신규 섹션으로 추가
2. 라우팅 정리
   - `/community`, `/dangol/feed` 진입 시 "소식" 탭이 활성화되도록 activeFor 갱신
   - `/benefits` 직접 진입은 유지하되 탭에서는 사라짐 → 마이 상단 카드에서 진입
3. 마이페이지 정보 구조 재정렬
   - 상단: 내 쿠폰 카드, 내 스탬프 카드 (가로 2분할)
   - 중단: 내 단골 가게 리스트
   - 하단: 알림 설정, 멤버십 관리, 계정 설정
4. 컨텍스트 배지
   - 모든 탭 최상단에 현재 상권 컨텍스트 배지(예: "고덕2동 골목형상점가") 노출

영향 화면
- TabBar.tsx, MyPage.tsx, BenefitsHub.tsx 진입 동선, RegionPortal/MarketHome 상단 배지

리스크
- 중간. 혜택 접근 경로 변경으로 단기 쿠폰 사용률 하락 가능 → 마이 상단 카드 강조와 토스트 안내로 보완

검증
- 소식 탭 진입률 ≥ 기존 "상점가 탭 > 커뮤니티/단골 소식 진입률" 합산의 1.2배
- 마이 상단 쿠폰 카드 클릭률 ≥ 기존 혜택 탭 클릭률의 0.8배

---

### 2-3. Phase 3 — 구조 정합화 (중장기, Option C 방향)

> 진행 상태: **1차 구현 완료 (2025-11)**. 하단 4탭 재편 / `MK-00` 단일 메인화 / 마이 상단 혜택 카드 / 헤더 상권 컨텍스트 칩 적용. 후속 항목(필터 강화·찜·다중 상권 시트)은 백로그.

목표
- 상권 컨텍스트를 모든 탭에서 잃지 않게 만들고, 가게/커뮤니티 탭을 단일 의도 단위로 단단히 한다.

작업 항목
1. **[구현완료]** 하단 4탭 재편: 홈 / 가게 / 커뮤니티 / 마이 (`src/components/TabBar.tsx`)
2. **[구현완료]** `MK-00 상권 홈` 단일 화면화 — 내부 main/stores/feed 탭 제거, 레거시 URL 자동 정리 (`src/screens/MarketHome.tsx`)
3. **[구현완료]** 마이 탭 상단 "내 혜택" 카드 + 운영 대시보드 시각 분리 (`src/screens/MyPage.tsx`)
4. **[구현완료]** 상단 헤더 상권 컨텍스트 칩 — 탭 시 `M-00`으로 이동해 상권 전환 (`src/components/AppHeader.tsx`, `src/App.tsx`)
5. **[백로그]** 가게 탭 필터/뷰 강화 — 업종/혜택유무/단골여부/영업중 필터, 지도 보기 토글, 즐겨찾기(찜) 도입 → 마이에 모아보기 섹션 신설
6. **[백로그]** 멀티 상권 사용자 전환 시트 — 컨텍스트 칩에서 시트로 직접 전환(현재는 `M-00` 경유)
7. **[백로그]** 운영자 모드 권한 기반 노출 강화 — 권한 없는 사용자에게 운영 카드 자체 숨김

영향 화면
- TabBar, StoresExplorer, StoreDetail, MyPage, RegionPortal, MarketHome

리스크
- 큼. 정보구조 변경 폭이 넓으므로 KPI 모니터링 + 단계 A/B 적용 필수

검증
- 가게 상세 진입 → 단골 등록/쿠폰 수령 전환율 상승
- 커뮤니티 탭 DAU/세션수 상승
- 상권 전환 액션이 발생하는 사용자 비율 측정 (멀티 상권 확장 의사결정 근거)

## 3. 네비게이션 외 UX 개선 백로그

각 항목은 단독으로 진행 가능하며, 위 Phase와 병행 가능하다.

### 3-1. 정보 계층(IA)
1. 모든 화면에 "현재 상권" 컨텍스트 배지 일관 노출
2. 가게 상세 내 정보 우선순위 재정의(영업 상태 → 혜택 → 메뉴 → 위치 → 리뷰 순)
3. 마이페이지 섹션 카드화 및 우선순위 재배치 (Phase 2 항목)
4. 검색 결과 화면 신설 검토 (가게/커뮤니티/이벤트 통합 검색)

### 3-2. 인터랙션
1. 하단 시트 표준화 (멤버십, 단골, 소식 작성 시트의 헤더·닫기·CTA 위치 통일)
2. 토스트 위치/지속시간 가이드 정리 (성공/경고/오류 톤 분리)
3. 빈 상태(Empty state) 일러스트/카피 표준화
4. 로딩/스켈레톤 적용 범위 정리 (현재 즉시 렌더링되는 화면이 많음)
5. 햅틱·트랜지션 가이드 (WebView 환경 가정)
6. 가게 카드 탭 영역 확대 및 보조 액션(찜, 공유) 노출 일관화

### 3-3. 정보 표현
1. 가격/할인 표기 규칙 통일 (원, 할인율, 기간 포맷)
2. 시간 표기 통일 (n분 전, n시간 전, 어제, 날짜)
3. 단골 수·멤버 수 등 카운트 표기 약식 규칙 (1.2K, 12K)
4. 카테고리 칩 색상 토큰 정리 (현재 4~5종 혼재)
5. 영업 상태 배지 색상/문구 통일 (영업중/곧 종료/영업종료)

### 3-4. 접근성
1. 색 대비 점검 (glass-strong 위 텍스트 명도비 4.5 이상)
2. 터치 타깃 최소 44pt 보장 (현재 일부 11px 텍스트 + small padding 카드 위험)
3. 폼 인풋 라벨/플레이스홀더 분리
4. 스크린리더 라벨 (탭바, 아이콘 버튼)
5. 다크 모드 토큰 준비 (Phase 3 이후 본격 적용)

### 3-5. 디자인 시스템
1. 토큰 정합화 — brand/ink/glass 변형 정리
2. 컴포넌트 카탈로그 보강 — Sheet, Toast, Badge, EmptyState, ContextChip 추가
3. 아이콘 세트 정리 — appIcons/storeIcons 분류 일관화
4. 카피 가이드 문서화 — 사용자 언어, 톤, 금지어 목록
5. iOS 글래스 효과 가이드 — backdrop-blur/opacity 사용 기준 명문화

### 3-6. 퍼포먼스/품질
1. 라우팅 동기화 시점 정리 (useEffect 다중 트리거 점검)
2. 이미지 lazy load 적용
3. 빈 데이터 시 placeholder 처리
4. 에러 바운더리 최소 1개 도입
5. 분석 이벤트 표준 스키마 정리 (trackEvent 호출부 점검)

### 3-7. 멤버십·전환 동선
1. 비멤버 락 패턴 통일 (현재 blur + CTA 위치가 화면마다 다름)
2. 멤버십 가입 동의 단계 축약 (현 다중 단계 검토)
3. 단골 등록 후 곧바로 단골 소식 구독 토글 노출
4. 쿠폰 수령 후 보유 쿠폰으로 자연 이동 유도
5. 쿠폰 사용 완료 후 가게 별점/후기 유도(선택)

### 3-8. 커뮤니티
1. 글쓰기 폼 카테고리 의무화 검토
2. 좋아요/댓글 상호작용 실제 동작 (현재 표시만)
3. 글 신고/숨기기 액션 추가
4. 멤버 프로필 라이트 페이지 (작성 글 모아보기)
5. 공지 핀 고정 기능

### 3-9. 단골 소식
1. 가게별 필터 + 카테고리 필터 동시 적용
2. 새 소식 배지(N) 표시
3. 알림 발송된 소식 식별 라벨
4. 소식 상세에서 가게 상세로 이동하는 단방향 동선 보강
5. 소식 작성 시 이미지 첨부(선택)

### 3-10. 운영자 영역(O-00)
1. 사용자 영역과 시각적 분리 (헤더 컬러, 상단 배너)
2. 작성 폼 UX 통일 (단골 소식, 쿠폰 발행, 알림 발송)
3. 데이터 카드 정렬 기준 노출 (오늘/이번주/이번달 토글)
4. 발행 후 결과 토스트 + 다시 보기 링크 표준화
5. 권한 안내 카피 정리 (가게/상권/지자체 권한 차이)

## 6. 검증 체크리스트 (Phase별 공통)

1. 변경 후 모든 화면에서 현재 컨텍스트(지역/상권)가 명확한가
2. 핵심 액션(쿠폰 수령/사용, 단골 등록, 글쓰기)의 클릭 깊이가 기존보다 늘지 않았는가
3. 비멤버 사용자의 차단 경험이 일관된가
4. 다크모드/접근성 토큰이 깨지지 않는가
5. URL과 화면 상태가 항상 동기화되는가 (뒤로 가기 포함)
6. 운영자 화면이 사용자 영역으로 새지 않는가
