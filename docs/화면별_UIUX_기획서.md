# 로컬마켓 — 화면별 UI/UX 기획서 (IA 리라이팅)

작성일자 : 2026-05-26  
문서 버전 : v1.0 (IA/URL/서비스기획 통합 반영)

> 본 문서는 최신 IA 및 URL 규칙, 멀티테넌트 서비스 전략을 기준으로  
> 로컬마켓의 사용자 화면과 운영자 화면을 재정의한 통합 UI/UX 기획서다.

---

## 0. 리라이팅 기준

1. 탐색 계층은 지자체 -> 상권 -> 점포 순서로 통일한다.
2. 지자체는 브랜딩명으로 노출한다.
- 강동 hero (regionSlug: gangdong-gu)
- 공주왕도심 (regionSlug: gongju-si)
- 익산문화거리 (regionSlug: iksan-si)
3. URL 컨텍스트는 slug 기반, 화면 텍스트는 브랜딩명 기반으로 분리한다.
4. G-00 전체 포털은 지자체 선택 전용 화면으로 정의한다.
5. 초기 1지역 운영 시에도 표준 진입은 G-00 선택 화면이며, 운영 배포에서만 강동 hero 지자체 포털을 첫 진입으로 둘 수 있다.

## 1. 서비스 화면 체계 (최신)

| 트랙 | 코드 | 화면명 | 핵심 역할 |
|---|---|---|---|
| Global | G-00 | 전체 포털 | 지자체 선택 허브 |
| Region | M-00 | 지자체 포털 | 지자체 단위 상권 목록/캠페인 |
| Market | MK-00 | 상권 포털 | 멤버십, 커뮤니티, 점포 탐색 |
| Store | ST-00 | 점포 상세 | 단골 등록, 점포 혜택, 단골 소식 진입 |
| User | U-00 | 개인 영역 | 멤버십, 단골, 쿠폰, 알림, 설정 관리 |
| Ops | O-00 | 운영자 영역 | 지자체/상권/점포/시스템 운영 |

## 2. 전역 진입 정책

### 2-0. 채널 정책

1. G-00 전체 포털은 웹 사이트에서만 노출한다.
2. 지역별 앱은 WebView 래퍼 앱으로 배포한다.
3. 지역별 앱의 시작 화면은 M-00 지자체 포털이다.
4. 즉, 앱은 지역별 다중 패키지, 서비스 코드는 단일 코드베이스를 유지한다.
5. G-00은 지자체 선택 전용이며 상권 홍보형 히어로를 두지 않는다.
6. M-00은 상단바 없이 지역 브랜드와 상권 카드를 중심으로 구성한다.

예시 시작 URL:
- 강동 앱: `/regions/gangdong-gu`
- 공주 앱: `/regions/gongju-si`
- 익산 앱: `/regions/iksan-si`

채널별 네비게이션 규칙:
- 웹: G-00 접근 가능, 지역 전환 UI 노출 가능
- 지역 앱(WebView): G-00 접근 비허용, 지역 전환 UI 기본 비노출
- 지역 앱(WebView)에서 M-00 이후 G-00으로 돌아가는 버튼/메뉴는 제공하지 않음
- 지역 앱(WebView) 홈 버튼은 항상 해당 지자체 M-00으로 복귀

### 2-3. 하단 네비게이션 기준

사용자 하단 네비게이션은 아래 매핑을 기준으로 고정한다.

1. 포털: M-00 지자체 포털
2. 상점가: MK-00 상권 포털
3. 혜택: 상권/점포 혜택 통합 허브(U-00-3)
4. 마이: 마이페이지/내 활동(U-00-1)

규칙:
- G-00은 지자체 선택을 위한 초기 진입 화면이다.
- M-00 진입 이후 하단 네비게이션으로 G-00 복귀 경로를 제공하지 않는다.

### 2-1. 루트 진입 정책

1. 1지역 운영 단계(MVP)
- `/` 진입 시 G-00 선택 화면을 노출한다.
- 권장: `/regions/gangdong-gu`를 시작점으로 고정

2. 다지역 운영 단계
- `/` 또는 `/regions` 진입 시 G-00 전체 포털 노출
- 사용자에게 지자체 선택을 먼저 요구

### 2-2. 브랜딩 표기 정책

| 구분 | 내부 식별자 | 사용자 노출명 |
|---|---|---|
| 강동구 권역 | gangdong-gu | 강동 hero |
| 공주시 권역 | gongju-si | 공주왕도심 |
| 익산시 권역 | iksan-si | 익산문화거리 |

원칙:
- 네비게이션, 카드, 헤더, 배지는 브랜딩명 우선
- 운영 로그, 권한 체크, API 호출은 내부 식별자 기준

## 3. 사용자 핵심 여정

```text
웹 진입
  -> G-00 전체 포털 (지자체 찾기)
  -> M-00 지자체 포털 (상권 선택)
  -> MK-00 상권 포털
      -> 멤버십 등록
      -> 커뮤니티/혜택 탐색
      -> ST-00 점포 상세
          -> 단골 등록
          -> 단골 소식 수신
  -> U-영역에서 내 활동/알림/쿠폰 관리
```

```text
지역 앱(WebView) 진입
  -> M-00 지자체 포털 (앱 바인딩 지역)
  -> MK-00 상권 포털
      -> ST-00 점포 상세
  -> 홈 버튼 탭 시 M-00 복귀
  -> G-00 이동 버튼은 노출되지 않음
```

보조 진입:
- QR/NFC: 해당 상권 또는 점포로 바로 진입
- 푸시 알림: 관련 컨텍스트 화면으로 딥링크

## 4. 화면별 상세 설계

### G-00. 전체 포털 (다지역 허브)

화면 목적:
- 사용자가 먼저 지자체를 선택하게 하는 글로벌 허브
- 계층 탐색의 1단계, 선택 전용 진입점

진입 경로:
- `/`
- `/regions`
- 딥링크 컨텍스트 없는 최초 실행

채널 제한:
- 웹 사이트에서만 노출
- 지역 앱(WebView)에서는 렌더링하지 않음

컴포넌트:
1. 지자체 선택 리스트
2. 선택된 지자체 요약 카드

상태 정책:
- 비로그인: 지자체 카드 + 공개 정보 노출
- 로그인/멤버십 1개: 내 상권 바로가기 우선
- 로그인/멤버십 다수: 최근 방문 상권 우선 정렬

전환:
- 지자체 카드 탭 -> M-00 지자체 포털

레이아웃 규칙:
- 히어로 배너를 두지 않는다.
- 하단 네비게이션을 두지 않는다.
- 지자체 선택 후 M-00에서만 지역성 콘텐츠를 확장한다.

---

### M-00. 지자체 포털

화면 목적:
- 지자체 단위 허브로 상권 목록, 지자체 캠페인, 공지 제공

URL 예시:
- `/regions/gangdong-gu` (강동 hero)
- `/regions/gongju-si` (공주왕도심)
- `/regions/iksan-si` (익산문화거리)

앱 네비게이션 기준:
- 지역 앱(WebView)의 기본 홈 화면
- 앱 로고/홈 탭/메인 복귀 액션은 모두 M-00으로 이동
- M-00 상단바는 노출하지 않음
- M-00 이후 G-00으로 이동하는 버튼/메뉴는 노출하지 않음
- 하단 "포털" 탭의 목적지는 G-00이 아니라 M-00으로 고정

컴포넌트:
1. 지자체 브랜딩 히어로: 브랜딩명, 소개 문구, 참여 지표
2. 참여 상권 목록
3. 지자체 통합 캠페인
4. 지자체 공지/리포트(공개 범위)
5. 상권 검색/필터(카테고리, 운영 상태)

핵심 UX 원칙:
- 상권 카드는 소개보다 현재 참여 가능한 액션(이벤트, 쿠폰, 투어)을 우선 노출
- 준비 중 상권은 회색 처리 + 오픈 알림 신청

전환:
- 상권 카드 탭 -> MK-00 상권 포털

---

### MK-00. 상권 포털

화면 목적:
- 멤버십 전환, 커뮤니티 활성화, 점포 탐색의 중심 화면

URL 예시:
- `/regions/gangdong-gu/markets/godeok-2dong`

핵심 컴포넌트:
1. 상권 히어로(대표 이미지, 멤버십 CTA)
2. 멤버십 상태 박스
- 비멤버: 멤버십 등록하기 강조
- 멤버: 내 멤버십 상태/활동 바로가기
3. 멤버 전용 커뮤니티 미리보기
4. 이벤트/쿠폰/스탬프투어 섹션
5. 추천 코스 섹션
6. 점포 목록(지도/리스트 토글)

탭 구조(쿼리):
- `?tab=main`
- `?tab=stores&view=map`
- `?tab=feed&feedTab=community`

전환:
- 멤버십 등록 -> MK-01
- 점포 카드 -> ST-00
- 커뮤니티 -> MK-02
- 혜택 -> U-00 (혜택 허브)

---

### MK-01. 멤버십 등록 플로우

목적:
- 비멤버를 1탭 구조로 멤버 전환

패턴:
1. 로그인 확인
2. 하단 시트 동의/안내
3. 등록 완료 피드백

UX 원칙:
- 가입 이점 4개 이내로 간결히 제시
- 정보 공유 항목은 명확히 고지
- 완료 후 커뮤니티 또는 원래 화면으로 복귀 선택 제공

---

### MK-02. 멤버 전용 커뮤니티 피드

목적:
- 상권 단위 폐쇄형 커뮤니티 운영

핵심 구성:
1. 커뮤니티/단골 소식 탭
2. 글쓰기 CTA(상단 + FAB)
3. 카테고리 필터
4. 피드 카드(공식/가게/멤버 뱃지)

권한:
- 멤버만 읽기/쓰기 가능
- 비멤버는 블러 + 멤버십 유도

---

### ST-00. 점포 상세

목적:
- 점포 방문 결정을 돕고 단골 전환을 유도

URL 예시:
- `/regions/gangdong-gu/markets/godeok-2dong/stores/godeok-bakery`

핵심 구성:
1. 점포 기본 정보
2. 단골 등록/상태 버튼
3. 단골 소식 미리보기
4. 점포 쿠폰/이벤트
5. 스탬프 적립 현황
6. 오시는 길

상태 정책:
- 비멤버: 단골 등록 비활성 + 멤버십 유도
- 멤버/비단골: 정보 받아보기 활성
- 단골: 단골 소식 보기 활성

전환:
- 단골 등록 -> ST-01
- 단골 소식 -> ST-02

---

### ST-01. 단골 등록 플로우

목적:
- 멤버 사용자를 점포 단골로 전환

패턴:
- 하단 시트 단일 단계 등록
- 등록 완료 후 점포 상세 복귀

핵심 안내:
- 단골 알림 수신
- 단골 전용 혜택
- 공유 정보 범위

---

### ST-02. 단골 전용 소식

목적:
- 점포 담당자가 단골에게 발송한 소식 확인

핵심 구성:
1. 단골 가게 필터(전체/가게별)
2. 최신순 소식 카드
3. 읽지 않음 뱃지

권한:
- 해당 가게 단골만 열람

---

### U-00. 개인 영역

하위 화면 묶음:
1. 내 활동
2. 알림함
3. 혜택 허브 (쿠폰/스탬프)
4. 추천 코스

---

#### U-00-1. 내 활동

목적:
- 멤버십/단골/알림 설정 통합 관리

URL 예시:
- `/me`
- `/me/memberships`
- `/me/dangol`
- `/me/settings`

구성:
1. 내 멤버십 목록(상권별)
2. 내 단골 가게 목록
3. 알림 수신 토글(상권별/가게별)
4. 해제 플로우(영향 안내)

---

#### U-00-2. 알림함

목적:
- 상권 및 점포 알림 확인과 즉시 이동

URL 예시:
- `/notifications`

딥링크 정책:
- 상권 이벤트 -> MK-00 이벤트 섹션
- 상권 쿠폰 -> U-00-3 혜택 허브
- 점포 소식 -> ST-02
- 스탬프 완주 -> U-00-3 또는 스탬프 상세

알림 도메인 규칙:
1. 멤버 알림: 상권 운영자 -> 상권 멤버십 가입자(시장 단위)
2. 단골 알림: 점포 운영자 -> 해당 점포 단골(점포 단위)
3. 상세 데이터/권한/검증 규칙은 `로컬마켓_서비스_운영_설계서` 2-3절(알림 도메인 경계)을 정본으로 따른다.

---

#### U-00-3. 혜택 허브 (쿠폰/스탬프)

목적:
- 쿠폰과 스탬프투어를 혜택 축으로 통합 제공

URL 예시:
- `/benefits`
- `/benefits?tab=stamp`
- `/benefits?tab=coupon&status=active`

구성:
1. 혜택 탭(스탬프/쿠폰)
2. 사용 가능/완료/만료 필터
3. 쿠폰 제시 화면(직원 확인 최적화)

---

#### U-00-4. 추천 코스

목적:
- 상권 체류 시간을 늘리는 큐레이션 동선 제공

구성:
1. 코스 카드 리스트
2. 코스 상세(동선/순서/가게 바로가기)
3. 스탬프투어 연계 CTA

---

## 5. 운영자 콘솔 (웹)

운영자 URL 네임스페이스:
- `/ops/**`

### O-00. 운영자 영역

하위 화면 묶음:
1. 지자체 운영 (구청/지자체)
2. 상권 운영 (상인회/운영사)
3. 점포 운영 (가게 담당자)
4. 시스템 운영 (플랫폼)

---

#### O-00-1. 지자체 운영

URL 예시:
- `/ops/regions/{regionSlug}/dashboard`
- `/ops/regions/{regionSlug}/reports`
- `/ops/regions/{regionSlug}/markets`

역할:
- 지자체 KPI 조회
- 상권별 성과 비교
- 지자체 통합 캠페인 운영

---

#### O-00-2. 상권 운영 (상인회/운영사)

URL 예시:
- `/ops/regions/{regionSlug}/markets/{marketSlug}/dashboard`
- `/ops/regions/{regionSlug}/markets/{marketSlug}/members`
- `/ops/regions/{regionSlug}/markets/{marketSlug}/feed`
- `/ops/regions/{regionSlug}/markets/{marketSlug}/campaigns`
- `/ops/regions/{regionSlug}/markets/{marketSlug}/notifications`

핵심 기능:

## 9. 최종 정리

1. 화면 탐색의 1단계는 지자체 선택이다.
2. 브랜딩명 노출과 내부 slug 식별을 분리해야 확장 시 혼선이 없다.
3. G-00은 다지역 허브, M-00은 지자체 운영 허브, MK-00은 전환 허브로 역할을 분리한다.
4. 핵심 전환은 멤버십 등록과 단골 등록이며, UI는 항상 이 두 행동을 짧은 경로로 유도해야 한다.
5. 지역 앱(WebView)에서는 M-00 이후 G-00으로 돌아가는 네비게이션을 제공하지 않는다.
