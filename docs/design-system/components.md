# 컴포넌트 규격 가이드

작성일자: 2026-05-26

---

## 1. 공통 원칙

1. 컴포넌트는 상태를 먼저 정의하고 스타일을 입힌다.
2. 동일 역할 컴포넌트의 높이, 타이포, 패딩은 일관 유지한다.
3. 채널 정책(web/app)은 컴포넌트 레벨에서 강제 가능해야 한다.

---

## 2. App Header

목적:
- 현재 컨텍스트와 핵심 액션(뒤로, 알림) 제공

필수 요소:
1. 좌측: 뒤로 버튼
2. 중앙: 화면 제목
3. 우측: 알림 버튼

상태:
- default
- transparent
- no-back

규칙:
- 앱 모드에서 홈 로고 탭이 있다면 목적지는 M-00만 허용
- 타이틀은 1줄 유지, 초과 시 말줄임
- G-00과 M-00은 상단바를 사용하지 않는 특수 화면으로 취급한다.

---

## 3. Tab Bar

목적:
- 핵심 1차 네비게이션

기본 탭:
1. 포털(M-00)
2. 상점가(MK-00)
3. 혜택(U-00-3)
4. 마이(U-00-1)

규칙:
- 포털 탭의 목적지는 G-00이 아닌 M-00으로 고정
- G-00은 초기 지자체 선택 전용이며, 하단 네비게이션 경로에 포함하지 않음
- G-00은 히어로 섹션 없이 선택 리스트만 노출한다.
- M-00은 하단 탭은 유지하되 상단바는 두지 않는다.
- 활성 탭은 색상 + 배경 + 아이콘 강조로 식별
- 탭 터치 영역은 최소 44x44 보장

---

## 4. 카드 컴포넌트

유형:
1. 지자체 카드
2. 상권 카드
3. 점포 카드
4. 피드 카드

규칙:
- 카드 상단 1영역에 핵심 CTA 1개만 우선 배치
- 메타 정보는 2줄 이내 유지
- 액션 우선순위는 가입/등록/참여/사용 순

---

## 5. Bottom Sheet

유형:
1. 멤버십 등록 시트
2. 단골 등록 시트

규칙:
- 핵심 메시지 1개 + 보조 설명 2개 이내
- 동의 체크/액션 버튼은 하단 고정
- 닫기 후 복귀 화면 컨텍스트 유지

---

## 6. Toast/피드백

목적:
- 성공/실패/안내 메시지 즉시 전달

규칙:
- 1~2초 내 핵심 의미 파악 가능해야 함
- 성공 토스트는 다음 행동 힌트 포함 권장
- 오류 토스트는 재시도/문의 액션 동반

---

## 7. 금지 규칙

1. 동일 화면에 1차 CTA 두 개를 동급으로 배치 금지
2. 브랜드 컬러 외 임의 포인트 컬러 남용 금지
3. 앱 모드에서 G-00 복귀 버튼 노출 금지
4. 텍스트 대비 부족한 유리 배경 위 연한 글자 사용 금지
