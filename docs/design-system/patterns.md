# 화면 패턴 및 안티패턴

작성일자: 2026-05-26

---

## 1. 핵심 사용자 여정 패턴

### 1-1. 웹 표준 여정
1. G-00 진입
2. M-00 선택
3. MK-00 탐색
4. ST-00 진입
5. U-00 관리

### 1-2. 앱(WebView) 표준 여정
1. M-00 진입
2. MK-00 탐색
3. ST-00 진입
4. 홈 액션 시 M-00 복귀

규칙:
- 앱 여정에서는 G-00 회귀 경로를 만들지 않는다.

---

## 2. 전환 패턴

### 2-1. 멤버십 전환
- 노출 위치: MK-00 상단
- 트리거: 커뮤니티/혜택 접근 전
- UI: 짧은 이점 설명 + 단일 CTA

### 2-2. 단골 전환
- 노출 위치: ST-00 중상단
- 트리거: 점포 상세 열람 후
- UI: 정보 수신 가치 + 동의 + 등록

---

## 3. 콘텐츠 패턴

1. 피드 카드
- 제목 1줄, 본문 2줄, 상태 배지

2. 혜택 카드
- 유효기간, 사용 조건, 즉시 사용 CTA

3. 운영 공지
- 중요도 배지 + 발행 주체 + 발행 시각

---

## 4. 안티패턴

1. 컨텍스트 없는 딥링크
- region/market/store 컨텍스트 없이 상세 진입 금지

2. 과도한 CTA 중첩
- 카드 하나에 버튼 3개 이상 배치 금지

3. 채널 정책 위반
- 앱에서 지역 전환 UI 상시 노출 금지
- 앱에서 루트 진입 후 G-00 노출 금지

4. 시각 계층 붕괴
- 동일 크기/동일 농도 텍스트를 반복 배치 금지

---

## 5. QA 체크 포인트

1. 여정 완결성
- 핵심 액션 이후 복귀 경로가 명확한가?

2. 정책 준수
- web/app 모드별 UI 분기가 정확한가?

3. 컨텍스트 유지
- 화면 이동 후 region/market/store가 유지되는가?

4. 접근성
- 대비, 터치 영역, 레이블이 최소 기준을 충족하는가?
